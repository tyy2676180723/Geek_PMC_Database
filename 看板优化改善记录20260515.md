## 1. 页面顶部间距缩小

**问题**：各看板标题距页面顶部间距过大，有效内容区域偏低。

**方案**：在 `streamlit_app.py` 注入全局 CSS，缩减 `.block-container` 的 `padding-top`。

```css
.block-container { padding-top: 1.5rem !important; }
```

## 2. 成品料号筛选失效修复

**文件**：`pages/shortage.py`

**问题**：在「待开工单缺料看板」选择成品料号后，下方数据框不响应，筛选结果为空。

**根因**：ERP 导出的成品料号列值有时是字符串 `"0"` 而非数字 `0`，直接 `> 0` 比较会失败。

**方案**：用 `pd.to_numeric(errors='coerce').fillna(0)` 转换后再比较。

```python
mask |= (pd.to_numeric(df_view[c], errors="coerce").fillna(0) > 0)
```

## 3. 筛选后 AgGrid 表格不更新修复

**文件**：`pages/shortage.py`

**问题**：修改筛选条件后，下方 AgGrid 表格仍显示旧数据，不重新渲染。

**根因**：AgGrid 组件的 `key` 固定，Streamlit 复用了旧组件缓存。

**方案**：用筛选后数据的行索引哈希值作为动态 key，数据变化时强制重建组件。

```python
_vk = abs(hash(tuple(df_view.index.tolist())))
editable_table(df_view[core_cols_exist], ["交期"], key=f"editor_core_{_vk}")
```

## 4. MRB 处理进度 GitHub 持久化

**文件**：`pages/mrb.py`、`requirements.txt`

**问题**：在 Streamlit Cloud 上保存「MRB缺料」处理进度后，容器重启即丢失（`git push` 无凭证）。

**方案**：使用 PyGithub 库通过 GitHub API 直接读写 `mrb_progress.json`，绕过 git 凭证限制。

- Streamlit Cloud Secrets 配置：`GITHUB_TOKEN = "github_pat_..."`
- Token 权限：仅 `Geek_PMC_Database` 仓库，Contents Read & Write
- 读取优先级：GitHub API → 本地文件
- 写入优先级：GitHub API → git CLI（本地运行兜底）

```python
repo = _gh_repo()
if repo:
    f = repo.get_contents(PROGRESS_GH_PATH)
    repo.update_file(PROGRESS_GH_PATH, msg, content, f.sha)
```

**处理进度推送状态展示**：保存后通过 `st.session_state` 持久存储推送结果与时间戳，页面重渲染后仍可见。

```python
st.session_state["save_status_a"] = {"ok": ok, "time": "2026-05-15 14:30:00"}
# 显示：✅ 已保存并推送到 GitHub（2026-05-15 14:30:00）
#    或：⚠️ 已本地保存，GitHub 推送失败 ——
```

## 5. 编辑单元格直接点保存数据丢失修复

**文件**：`utils.py`

**问题**：在 AgGrid 单元格编辑状态下直接点击「保存」按钮，当前正在编辑的值不会提交。

**根因**：点击按钮时单元格仍处于编辑模式，值未写入 AG Grid 的数据模型。

**方案**：在 `_GRID_BASE` 中加入 `stopEditingWhenCellsLoseFocus: True`，焦点离开单元格时自动提交。

```python
_GRID_BASE = dict(
    ...
    stopEditingWhenCellsLoseFocus=True,
)
```

## 6. 列宽自适应覆盖全部列（含屏幕外）

**文件**：`utils.py`

**问题**：点击列菜单「所有列自适应」，只有当前可见列生效，横向滚动后的列宽不变。

**根因**：AG Grid 默认开启列虚拟化，仅渲染可视区域内的列，`autoSizeAllColumns()` 无法处理未渲染的列。

**方案**：在 `_GRID_BASE` 中加入 `suppressColumnVirtualisation: True`，强制渲染所有列。

```python
_GRID_BASE = dict(
    ...
    suppressColumnVirtualisation=True,
)
```

同时使用 `IntersectionObserver` 在 tab 切换后 grid 进入视口时自动触发 `autoSizeAllColumns()`。

## 7. AgGrid 最后一行被遮挡修复

**文件**：`utils.py`

**问题**：数据框滚动到底部时，最后一行只显示一半，鼠标滚轮无法完整显示，键盘方向键可以。

**根因**：横向滚动条使用 `position: fixed; bottom: 0` 固定在 iframe 底部，脱离 flex 文档流后叠在最后一行上方，AG Grid 内部的 `maxScrollTop` 计算偏小。

**方案**：在 AG Grid 底部添加一个 **28px 的幽灵钉住行（Pinned Bottom Row）**，将最后一行数据顶到横向滚动条上方。

```python
# utils.py — _build_go 函数
go["pinnedBottomRowData"] = [{}]           # 空数据行
go["getRowHeight"] = _PINNED_ROW_HEIGHT_JS # pinned 行高 28px

# _ON_READY_JS 中注入 CSS 隐藏幽灵行
st.textContent = '.ag-floating-bottom { opacity: 0 !important; pointer-events: none !important; }'
```

幽灵行特性：
- 不参与排序、筛选
- `editable_table` 返回的 `result["data"]` 不含幽灵行，无需过滤
- CSS 隐藏后用户完全不可见、不可交互

> 详细校准方法见：[AgGrid最后一行显示问题解决方案.md](AgGrid最后一行显示问题解决方案.md)

## 8. MRB 处理进度同步到本地生成文件

**文件**：`MRB/mrb_shortage.py`

**问题**：本地运行 `mrb_shortage.py` 生成 `MRB缺料.xlsx` 时，看板上已保存的「处理进度」列不会自动回填。

**方案**：`mrb_shortage.py` 运行时自动 `git pull` 看板目录，读取最新 `mrb_progress.json`，在写入 Excel 时将处理进度一并填入对应物料行。

```python
progress = load_dashboard_progress(base_dir)
write_sheet(ws1, work_order, "工单缺料汇总", progress.get("工单缺料", {}))
```
